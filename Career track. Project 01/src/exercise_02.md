# Exercise 02. SWOT-анализ навыков

Анализ выполнен на основе результата `exercise_01.md` (анализ 7 вакансий на российском рынке для роли Junior QA Engineer) и колеса баланса из `Карьерный трек. Проект 00`. Целевой рынок — РФ.

> ⚠️ MIRO рекомендован как инструмент в README. Здесь воспроизвожу классическую SWOT-сетку в Markdown 2×2, чтобы артефакт был частью репозитория и пригоден для P2P.

---

## Классическая SWOT-сетка 2×2

```
┌──────────────────────────────────────┬──────────────────────────────────────┐
│ STRENGTHS (Сильные стороны)          │ WEAKNESSES (Слабые стороны)          │
│ — внутренние, под моим контролем     │ — внутренние, под моим контролем     │
├──────────────────────────────────────┼──────────────────────────────────────┤
│ OPPORTUNITIES (Возможности)          │ THREATS (Угрозы)                     │
│ — внешние, использовать              │ — внешние, реагировать               │
└──────────────────────────────────────┴──────────────────────────────────────┘
```

| | **Полезно для цели** | **Вредно для цели** |
|---|---|---|
| **Внутренние факторы (от меня)** | **Strengths** (см. §1) | **Weaknesses** (см. §2) |
| **Внешние факторы (среда)** | **Opportunities** (см. §3) | **Threats** (см. §4) |

## 1. Strengths — сильные стороны

| # | Сильная сторона | Почему это сильная сторона |
|---|------------------|----------------------------|
| 1 | **Системное обучение в «Школе 21»** | Подтверждённый pet-практикой опыт по теории тестирования, баг-репортам, Chrome DevTools, HTTP/HTTPS, cookies (см. соседние проекты `Дефекты/`, `Тестирование web/`). Это уже закрывает базу 5/7 вакансий из `exercise_01.md`. |
| 2 | **Привычка к peer-review и P2P-коммуникации** | Школа формирует устойчивые soft skills: умение задавать вопросы, объяснять решения, принимать критику. Это сильно отличает выпускников от самоучек. |
| 3 | **Структурированный подход к задачам** | Привычка декомпозировать сложные задачи (видно по структуре сданных проектов: чек-листы, таблицы, чёткие выводы). Это ценится QA-командами на собеседовании. |
| 4 | **Аналитический склад ума и внимание к деталям** | Базовая профессиональная черта для QA — есть, подтверждается баг-репортами и SWOT-анализом, который сам же составляю. |
| 5 | **Ресурсы:** время (20-25 ч/неделю на обучение), доступ к учебной инфраструктуре «Школы 21», Rocket.Chat-сообщество, бесплатные открытые курсы | Из колеса баланса. Можно конвертировать в навыки. |
| 6 | **Достижения на текущий момент:** 4 завершённых учебных проекта (QA-направление, Career-track 00) + начатый репозиторий с проектами на GitHub | Уже есть, что показать рекрутеру помимо «учится в Школе 21». |
| 7 | **Базовая дисциплина регулярного обучения** | Подтверждается тем, что проекты сдаются последовательно, без длительных пауз. Это редкий ресурс на собеседовании джунов. |

### Что говорят окружающие (сильные стороны по мнению других)

- *«Хорошо структурирует мысли»* — обратная связь от пиров на P2P-защитах.
- *«Не боится задавать вопросы и докапываться до сути»* — типичный фидбек на peer-review.
- *«Чётко оформляет отчёты, сразу понятно, что и где сделано»* — обратная связь по сданным проектам.

## 2. Weaknesses — слабые стороны

| # | Слабая сторона | Как проявляется | Какие задачи мешает делать |
|---|----------------|------------------|----------------------------|
| 1 | **Нет коммерческого опыта** | Все 7 проанализированных вакансий хотят либо «опыт от 6 мес», либо сильное портфолио. Сейчас у меня учебные проекты, но нет работающего продакшен-кода. | Откликаться на «классические» junior-позиции — низкая конверсия. |
| 2 | **Python для автотестов — на нуле** | Не написан ни один автотест на Pytest/Playwright, не освоены фикстуры, моки. | Любые позиции с автоматизацией (5/7 вакансий). |
| 3 | **Английский на уровне A2-B1** | Читаю официальную документацию со словарём. Не могу свободно общаться на стендапе. | Международные проекты внутри РФ-компаний (Яндекс, Avito), повышение в Middle/Senior. |
| 4 | **Postman — только теория** | Не строил коллекций, не запускал Newman в CI, не делал авторизацию по токенам. | Любые позиции, где есть REST API (6/7 вакансий). |
| 5 | **Прокрастинация при сложных/долгих задачах** | На длинных проектах есть периоды снижения темпа, особенно на этапе «всё кажется непонятным». | Подготовка к экзаменам, освоение нового стека. |
| 6 | **Слабый network в IT** | Нет ментора, мало знакомых senior-QA, никто внутри целевых компаний не порекомендует. | Сложнее получить «тёплый» оффер, приходится опираться только на холодные отклики. |
| 7 | **Низкая стрессоустойчивость в момент технического собеседования** | Под давлением могу «забыть» базовые вещи. | Первые 5-10 собеседований будут проходить неуверенно — это нужно отрепетировать. |

## 3. Opportunities — возможности (внешние)

| # | Возможность | Как использую |
|---|-------------|----------------|
| 1 | **Активный найм QA в РФ.** По данным hh.ru в 2026 году QA — стабильный топ-10 IT-вакансий. | Откликаюсь регулярно, рынок сам толкает к собеседованиям. |
| 2 | **Программы стажировок крупных компаний** (Avito, Яндекс, Сбер, Ozon, VK). | Подаюсь на все весной/осенью, они принимают студентов «Школы 21» с открытыми руками. |
| 3 | **One Day Offer от Сбера для QA** — быстрый формат, отбор за 1 день. | Зарегистрироваться на ближайший, получить опыт и фидбек, даже если не пройду. |
| 4 | **Бум AI-инструментов для тестировщиков** (Cursor, ChatGPT, Claude, генерация тест-кейсов). | Использую при обучении и потом на работе — это ускоряет освоение Python в разы и выделяет меня среди джунов, которые этого ещё боятся. |
| 5 | **Сообщество «Школы 21»** | Можно найти пиров, которые уже работают в QA, попросить ревью резюме, провести mock-собеседование. |
| 6 | **Открытые публичные тестовые стенды** (saucedemo.com, httpbin.org, JSONPlaceholder) | Закрывают пробел «нет где практиковать API/UI-автотесты». |
| 7 | **Митапы и конференции онлайн** (Heisenbug, QA Talks) — много записей в открытом доступе. | Бесплатный апскил + темы для small talk на собеседовании. |
| 8 | **Тренд импортозамещения IT** — компании активно набирают своих специалистов и охотнее берут джунов с потенциалом. | Целюсь в Яндекс/Сбер/VK/Ozon, у которых сильные внутренние школы QA. |

### Уроки из чужих ошибок

- *Не идти в первый попавшийся аутсорс* — джуны там за полгода-год выгорают и теряют скорость роста (отзывы из QA-чатов).
- *Не откликаться без подготовки резюме под конкретную вакансию* — рекрутер видит шаблонный текст и отказывает на этапе скрининга.
- *Не игнорировать английский «на потом»* — на грейде middle/senior это становится потолком.

## 4. Threats — угрозы (внешние)

| # | Угроза | Как влияет | Как буду реагировать |
|---|--------|-----------|----------------------|
| 1 | **Сильная конкуренция на junior-позиции в РФ.** На одну вакансию приходит 200-300 откликов. | Снижает вероятность приглашения на собеседование. | (а) Подаюсь на 5-10 вакансий в день; (б) делаю сопроводительные письма под каждую; (в) развиваю pet-портфолио — выделяюсь среди шаблонных откликов. |
| 2 | **Тренд автоматизации тестирования** — компании уменьшают долю чисто ручных QA. | Если не освою Python+автотесты за 6-12 месяцев, рынок «съест» позиции, на которые я могу претендовать. | План закрытия пробелов из `exercise_01.md`, §3 — приоритет №1 на ближайшие 3 месяца. |
| 3 | **Макроэкономическая нестабильность** (курс рубля, инфляция, заморозка найма в отдельных индустриях). | Может временно сократить найм в банках/ретейле. | Диверсифицирую цели: смотрю и продуктовые компании, и B2B, и финтех. |
| 4 | **Тренд на «эффективность» — сокращение HR и долгие циклы найма.** | Собеседование может идти 1,5-2 месяца. | Не ждать одного оффера, а вести параллельно 5-10 воронок. |
| 5 | **Зарубежные сервисы и продукты уходят** — меньше международного опыта в РФ. | Снижается ценность английского в моменте, но в долгосроке — наоборот, повышается. | Не сбавляю темп с английским, готовлюсь на длинной дистанции. |
| 6 | **Слабые стороны (см. §2) могут затормозить движение к цели**, если их не закрывать активно. | Конкретно: без Python + опыта — буду идти на грейд middle 3 года вместо 1,5-2. | План закрытия + ежемесячная самопроверка по чек-листу из `Карьерный трек. Проект 00/src/exercise_01.md`. |
| 7 | **AI частично «съедает» ручное тестирование** — генерация тест-кейсов, авточек по требованиям. | На горизонте 5 лет роль чистого ручного тестировщика сокращается. | Сразу делаю акцент на автоматизации + AI-инструментах, а не «ручник навсегда». |

## 5. Стратегические выводы из SWOT

Сопоставляем квадранты, чтобы понять, на чём фокусироваться.

| Сценарий | Что делать |
|----------|-----------|
| **S × O** (использовать сильные стороны для возможностей) | Использовать школьную дисциплину обучения + AI-инструменты для быстрого освоения Python+Pytest. Подаваться на стажировки Яндекса/Avito/Сбера, опираясь на уже сделанные проекты в репозитории. |
| **W × O** (использовать возможности для закрытия слабых сторон) | Стажировки закроют «нет опыта» и «слабый network» одновременно. Бесплатные open-source стенды и AI закроют «нет Python/Postman». |
| **S × T** (использовать сильные стороны против угроз) | Системность и структурность → выделяться среди потока откликов сопроводительными письмами + pet-проектами. Это снижает риск «затеряться в 200+ откликах». |
| **W × T** (опасная зона: слабые стороны × внешние угрозы) | Английский A2 + тренд на автоматизацию + сокращение ручного QA = потолок роста через 2-3 года. **Это приоритет №1: английский + Python в ближайшие 6 месяцев — без компромиссов.** |

---

## 6. Главный фокус на ближайшие 3 месяца (по итогам SWOT)

1. **Закрыть W2 (Python для автотестов)** — ежедневно 1-2 часа.
2. **Закрыть W4 (Postman практика)** — за 2 недели.
3. **Параллельно W3 (английский)** — 30 минут Duolingo + 1 разговорный клуб в неделю.
4. **Использовать O2 (стажировки)** — подаюсь на ближайший набор Avito/Яндекса.
5. **Использовать O5 (network школы)** — нахожу 2-3 пиров с опытом QA, прошу провести mock-собеседование.

После 3 месяцев — переоценка SWOT.
