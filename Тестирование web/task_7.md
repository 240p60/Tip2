## 3. Приоритезация ошибок

Ошибки разнесены по уровню воздействия на пользователя, доступность и SEO. Каждой группе соответствует разумный SLA исправления.

### 🔴 Высокий приоритет (исправить в текущем спринте)

| Ошибка | Сколько | Почему критично |
|--------|---------|-----------------|
| `An img element must have an alt attribute` | 4 | Прямое нарушение WCAG 2.1 (AA): пользователи с screen reader не получают информацию. Удар по a11y и SEO. |
| `Duplicate ID` | 8 | Скрипты, обращающиеся к `getElementById()`, могут работать некорректно; ломает JS-логику. CSS-селекторы по id могут давать непредсказуемые стили. |
| `Element X not allowed as child of Y` (например, `<div>` внутри `<button>`) | 5 | Невалидный DOM, часть браузеров «починит» дерево сама и сломает обработчики событий. |
| `Stray end tag / Unclosed element` | 6 | Гарантированно ломает layout в части браузеров (особенно Safari/Firefox). |
| Отсутствие `<title>` или дубликаты `<title>` на разных страницах (если выявлено) | — | Удар по SEO. |

### 🟡 Средний приоритет (в течение 2 ближайших спринтов)

| Ошибка | Сколько | Почему важно |
|--------|---------|--------------|
| `Section lacks heading` | 3 | Снижает структурированность для SEO и screen reader, но не ломает интерфейс. |
| `aria-*` атрибуты с невалидными значениями | 2 | Влияют только на пользователей screen reader, но это всё ещё a11y. |
| `Attribute X not allowed on element Y` (например, `placeholder` на `<select>`) | 5 | Атрибут игнорируется, поведение остаётся, но валидность HTML страдает. |
| `Bad value for attribute … on element …` (например, неверный `type` у скрипта) | 2 | Браузер чаще всего обработает, но SEO-парсеры могут пометить страницу как «грязную». |

### 🟢 Низкий приоритет (бэклог, отдельный таск на чистку)

| Ошибка / предупреждение | Сколько | Почему можно отложить |
|------------------------|---------|-----------------------|
| `Trailing slash on void elements has no effect and interacts badly with unquoted attribute values` (`<img />` в HTML5) | 12 | Стилистическая придирка; на отображение не влияет. |
| `Consider adding a lang attribute …` (для вложенного блока) | 2 | Один `lang="ru"` уже стоит на `<html>` — внутренний инлайн-блок не критичен. |
| Лишние пустые атрибуты (`class=""`) | 3 | Не ломают ничего, чисто косметика. |
| Использование устаревших атрибутов (`bgcolor`, `border` на таблицах) | 1 | На сайте практически отсутствует, можно вычистить в плановом рефакторинге. |

## 4. Что сделать команде по итогам валидации

1. **Завести 3 задачи** соответствующего приоритета в трекер: «Чистка Duplicate IDs», «Добавить alt всем `<img>`», «Закрыть незакрытые теги в шаблонах».
2. **Включить W3C-валидацию в CI** (например, через `html-validate` или `htmlhint`) — большая часть ошибок ловится автоматически перед мержем.
3. **Подключить axe DevTools / Lighthouse-аудит** — это покроет блок a11y, который W3C-валидатор покрывает лишь частично.
4. **Сделать e2e-тест на ключевые сценарии** (форма заявки, мобильное меню) — валидатор ловит синтаксис, но не функциональность.
5. **Завести в шаблоне баг-репорта** поле «приоритет на основе W3C-валидации», чтобы команда говорила на одном языке.

### Прикладной вывод

Сайт «Школы 21» проходит валидацию с замечаниями, среди которых нет критичных для отображения, но есть существенные для **доступности и SEO**. Главные «болевые точки» (Duplicate IDs, отсутствие alt, незакрытые теги) лечатся за один-два спринта и сильно поднимают качество кода.
