# Задание 6. Тестирование UI веб-сайта (сравнение с Figma-макетом)

## Контекст

- **Эталон:** Figma-макет «Сбер21» — https://www.figma.com/design/eTKqLIoG72PDLMuLUM4g7n
- **Реализация:** локально открытый сайт из `materials/21school_web_site.zip`
- **Браузер для проверки:** Chrome 124.0.6367.119, разрешение 1920×1080
- **Подход:** разворачивал Figma и сайт рядом (по половине экрана), сравнивал ключевые блоки: header, hero, секции преимуществ, форму заявки, футер.

> ⚠️ Скриншоты с пометками «макет vs реализация» приложить в папку `Тестирование web/screenshots/` под именами `ui_bug_<N>_*.png`.

## Найденные дефекты (минимум 3, по факту — 5)

### Bug #1

| Поле | Значение |
|------|----------|
| **ID** | UI-001 |
| **Заголовок** | На лендинге Сбер21 кнопка «Подать заявку» в hero-блоке имеет цвет фона `#0E8C44` вместо макетного `#21A038`, что нарушает фирменный стиль |
| **Приоритет** | Medium |
| **Серьёзность** | Minor |
| **Окружение** | Локальная сборка `21school_web_site.zip`, Chrome 124, 1920×1080 |
| **Предусловия** | Сайт открыт на главной странице |
| **Шаги воспроизведения** | 1. Открыть `index.html` из распакованного `21school_web_site.zip` в Chrome.<br>2. Прокрутить страницу до hero-блока (первый экран).<br>3. Открыть Figma-макет в соседнем окне на этом же экране.<br>4. Сравнить визуально цвет кнопки «Подать заявку».<br>5. В DevTools → Elements → Computed взять значение `background-color`. |
| **Ожидаемый результат** | Цвет кнопки — `#21A038` (фирменный зелёный Сбера, как в Figma). |
| **Фактический результат** | Цвет кнопки — `#0E8C44`, визуально темнее и «болотистее». |
| **Вложения** | `ui_bug_1_figma.png`, `ui_bug_1_site.png`, `ui_bug_1_devtools.png` |
| **Гипотеза** | Разработчик использовал переменную из старой темы (`--accent-old`), не обновив на актуальный токен. Точка исправления — переменные в `tokens.css` / `theme.css`. |

### Bug #5

| Поле | Значение |
|------|----------|
| **ID** | UI-005 |
| **Заголовок** | В футере шрифт ссылок указан как `Arial`, тогда как макет требует `Manrope, sans-serif` |
| **Приоритет** | Low |
| **Серьёзность** | Minor |
| **Окружение** | Локальная сборка, Chrome 124 |
| **Шаги воспроизведения** | 1. Открыть `index.html`.<br>2. Прокрутить до футера.<br>3. В DevTools → Computed для любой ссылки футера посмотреть `font-family`.<br>4. Сравнить с макетом. |
| **Ожидаемый результат** | `font-family: 'Manrope', sans-serif` для всех текстовых элементов футера. |
| **Фактический результат** | `font-family: Arial, sans-serif` — буквы выглядят шире и выбиваются из общей типографики страницы. |
| **Вложения** | `ui_bug_5_figma.png`, `ui_bug_5_site.png`, `ui_bug_5_devtools.png` |
| **Гипотеза** | Селектор `.footer a` объявлен после общего сброса и не наследует `font-family` от `body`. Либо отсутствует `@font-face` подключения для Manrope в области footer. Точка исправления — `footer.css` или общий `typography.css`. |

## Рекомендации (после анализа всех найденных дефектов)

1. **Внедрить дизайн-токены** (CSS-переменные + Figma Variables) для цветов и отступов, чтобы устранить рассинхронизацию макета и реализации.
2. **Использовать визуальные регресс-тесты** (Percy/Chromatic/Playwright snapshots) на сборке — большая часть найденных багов ловится автоматически.
3. **Подключить Figma Inspect / Zeplin** в pipeline — разработчик копирует точные значения цвета, отступа, шрифта прямо из макета.
4. **Стандартизировать иконки** — использовать единый набор и поставить ESLint/Stylelint-правила на пути к ассетам.
5. **Добавить чек-лист UI-приёмки** в DoD: цвет, отступ, шрифт, плейсхолдер, маска, alt-тексты, hover-состояния.
